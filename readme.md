> One topic a day. One commit a day. No excuses.

A 30-day, hands-on journey through Python — from `print("Hello, World!")` to building a small full project. Every day lives in its own folder with notes, exercises, and solutions. This repo is both my personal log and a resource anyone can fork and follow along with.

---

## 📌 About This Challenge

I'm a developer,just starting out with Pytho. I have learnt that the best engineers never stop sharpening the fundamentals — and Python rewards revisiting the basics with fresh eyes. This challenge is my commitment to:

- **Consistency** — show up and write code every single day for 30 days straight.
- **Depth over speed** — each day includes real exercises, not just reading.
- **Public accountability** — daily commits, visible progress, no skipped days.
- **Build, don't just learn** — the back half of the challenge is project-driven.

If you're following along, fork this repo, star it, and tag your own progress with `#30DaysOfPython`.

---

## 🧭 How It Works

1. Each day has a folder: `day_1/`
2. Every folder contains:
   - `exercises.md` — practice problems for that day
   - `solutions.py` — my worked solutions
3. Progress is tracked in the table below and updated daily.
4. Each day gets its own commit, ideally with a clear message like `Day 07: Sets & Frozensets`.

### Rules I'm Holding Myself To

- ⏱️ Minimum 1 hour of focused coding per day.
- 🚫 No skipping — if a day is missed, it's made up before moving forward, not skipped.
- 📝 Every day ships with at least one runnable exercise file.
- 🔁 Concepts from earlier days get reused in later exercises (spaced repetition).

---

## 🛠️ Prerequisites

- Basic command-line comfort (navigating folders, running scripts)
- [Python 3.11+](https://www.python.org/downloads/) installed
- A code editor (I'm using [VS Code](https://code.visualstudio.com/))
- Git & GitHub basics

### Setup

```bash
# Clone the repository
git clone https://github.com/UdohLawrence/30-days-of-python.git
cd 30-days-of-python

# (Optional) Create a virtual environment
python -m venv venv
source venv/bin/activate      # macOS/Linux
venv\Scripts\activate         # Windows

# Install dependencies (added as the challenge progresses)
pip install -r requirements.txt
```
